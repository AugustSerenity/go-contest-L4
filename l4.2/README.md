### Распределённая CLI-утилита (grep/cut/sort) с кворумом и concurrency
### [Задание](./docs/task.md) 


`mycut` — это распределённая реализация Unix-утилиты cut с поддержкой многосерверной обработки данных и передачи результатов по сети.

Утилита может работать в двух режимах:

`server` — обрабатывает данные (cut) и отправляет координатору

`coord `— собирает данные от всех серверов, объединяет и сортирует

### Установка
```sh
make build
```

### Запуск в обычном (локальном) режиме
```sh
echo -e "a\tb\tc" | ./mycut --mode=server --f="1,3"
```
Вывод:
```sh
a   c
```
Разделитель по умолчанию: `\t`

### Пример распределённого использования (3 сервера + координатор)
Можно проверить в ручную как показано ниже или использовать автотест из пункта `Тест (сравнение с GNU cut)`

***ручная проверка***

Создадим файлы:

```sh
echo -e "a\tb\tc\nd\te\tf\ng\th\ti" > data1.txt
echo -e "1\t2\t3\n4\t5\t6\n7\t8\t9" > data2.txt
echo -e "x\ty\tz\np\tq\tr\nu\tv\tw" > data3.txt
```

Запускаем координатор(Терминал №1):
```sh
./mycut --mode=coord \
    --addr="localhost:9100" \
    --peers="localhost:9101,localhost:9102,localhost:9103" \
    --f="1,3"
```
Запускаем серверы(Терминал №2, Терминал №3, Терминал №4):
```sh
./mycut --mode=server \
    --addr="localhost:9101" \
    --peers="localhost:9100" \
    --f="1,3" < data1.txt

./mycut --mode=server \
    --addr="localhost:9102" \
    --peers="localhost:9100" \
    --f="1,3" < data2.txt

./mycut --mode=server \
    --addr="localhost:9103" \
    --peers="localhost:9100" \
    --f="1,3" < data3.txt
```
Координатор выводит объединённый и отсортированный результат:
```sh
1    3
4    6
7    9
a    c
d    f
g    i
p    r
u    w
x    z
```
### Тест (сравнение с GNU cut)
Запуск
```sh
make run
```
В проекте есть файл `test.sh.`

Он автоматически проверяет:
- корректность cut
- корректность распределённого режима
- соответствие GNU cut
